# NoteFragments 操作方法・備忘録・仕様書

このファイルは、操作方法の説明、開発中のメモ、仕様の整理をひとつにまとめるための追記用ドキュメントです。
思いついた内容を順番に足していく前提で、軽く書き足せる形にしています。

## 1. 目的

- NoteFragments の使い方をまとめる
- 開発中に気づいたことを残す
- 仕様の前提や制約を整理する
- 後から見返しても迷わないようにする

## 2. 基本操作

### 2.1 メモを保存する

- 文章を入力して `Save Note` を押す
- `Ctrl + Enter` でも保存できる
- `Enter` 単体は改行のまま

### 2.2 現在のページを保存する

- `Save URL` を押す
- 現在のタブの URL とタイトルを保存する

### 2.3 保存先を切り替える

- `Save to` のプルダウンで保存先ブランチを選ぶ

### 2.4 タイムラインを見る

- 下の `Fragments` に保存済みフラグメントが表示される
- `All Fragments` で表示を戻せる
- `Text` / `Image` / `URL` で絞り込める

### 2.5 ドキュメントを開く

- `Open Builder` を押すと文書編集画面が開く
- 文書画面では保存済みフラグメントを組み合わせて Markdown を作れる

### 2.6 文書を書き出す

- `Document Builder` で `Export .md` を押す
- OS の保存ダイアログが開くので、保存先を選ぶ
- 1 回選んだ保存先は、次回以降も同じダイアログ経路で開きやすい
- `Save as` をキャンセルした場合は保存されない
- 保存できたら、実際に `.md` ファイルが作成される

### 2.7 画像を単体保存する

- サイドバー上の画像フラグメントにある `Save image` を押す
- OS の保存ダイアログが開くので、保存先を選ぶ
- 保存するときは、表示用の軽い画像ではなく、元画像を保存する
- 画像ファイルは `.md` と同じフォルダに置かなくてもよい
- 文書の export 時は、画像がある場合に `NFAssets` フォルダへまとめて出力する

## 3. 開発メモ

- 開発確認は Vivaldi を使う
- Chromium 系ブラウザでの動作を優先する
- Firefox 専用の挙動は初期前提にしない
- `Ctrl + Enter` 保存はメモ入力欄だけに効かせる

## 4. 仕様メモ

- 保存データは `browser.storage.local` を使う
- フラグメントは `text` / `image` / `url` を基本にする
- 既定の保存先は `Inbox`
- 右クリックメニューから選択テキストや画像を保存できる
- 拡張アイコンでサイドパネルを開閉できる

### 4.1 画像ファイル名のルール

- `NFAssets` に入れる画像は、できるだけ衝突しない名前にする
- 基本は `img-<hash>.<ext>` 形式にする
- `<hash>` は画像の元URLや元データから作る短い識別子にする
- `<ext>` は元の拡張子か MIME type に合わせる
- 例: `img-1a2b3c4d5e6f.png`
- 同じ画像なら、できるだけ同じ名前になるようにする
- 文書ごとに別名へ変えず、ワークスペース内では再利用しやすくする

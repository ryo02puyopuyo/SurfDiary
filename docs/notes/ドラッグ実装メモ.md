# ドラッグ実装メモ

## 何がうまくいかなかったか

- ブラウザ任せの `dragstart / dragover / drop` は、Vivaldi のサイドパネルでは安定しなかった
- `setDragImage()` だけでは、プレビューが出てもカーソル追従が見えなかった
- `drop` イベントが入力欄や内部要素に吸われると、合体も挿入も発火しなかった
- `getBlockById()` のような基本関数が抜けると、ドラッグは始まっても判定が全部失敗した
- 前回のドラッグ状態を消し忘れると、色だけ残って次の操作が壊れた

## どう直したか

- 最終的に `pointermove / pointerup` ベースの自前追従に切り替えた
- 追従中のプレビューは `transform` で動かした
- 落とし位置の中央判定で「合体」と「挿入」を分けた
- 見た目と判定を同じコードで管理することで、サイドパネルでも安定しやすくした

## 学び

- サイドバーのような特殊な UI では、ブラウザ標準 DnD に頼りすぎないほうが安定する
- まずは「動く見た目」を自前で作り、その上に「合体」「挿入」の判定を乗せるほうが分かりやすい
- 状態解除は必ず 1 か所にまとめる

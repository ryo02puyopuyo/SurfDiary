# SurfDiary プロジェクト・ビジョン＆ロードマップ

このドキュメントは、ブラウザ拡張機能 **SurfDiary (サーフダイアリー)** のコンセプト、現在の進捗、および将来の発展方向を一つにまとめたものです。

---

## 1. コンセプト：Web体験をそのまま日記に
SurfDiaryは、ネットサーフィン（Surf）の体験を途切れさせることなく、気になった瞬間を「パンくず」のように残し、それを後に「日記（Diary）」へと編み上げることができるツールです。

- **シームレス**: 右クリック一つで画像やテキストを保存。
- **断片化 (Fragment-based)**: 保存した要素はすべて独立した「断片」として扱い、何度でも再利用可能。
- **Markdown中心**: 汎用性の高いMarkdown形式を採用し、ブラウザの外でも自分のデータとして自由に扱える。

## 3. 実装ロードマップ

### フェーズ0：データ基盤の固定
最初に、今後の機能追加で壊れないように保存形式を決めます。

- **データモデル定義**: `Fragment`, `Branch`, `Document` の型をコード上で明文化する。
- **保存層の抽象化**: `browser.storage.local` を直接触らず、`saveBlock` / `loadBlocks` のような薄いAPIを挟む。
- **既存メモの移行方針**: いまの `memos` 配列を、新しい `Fragment` 形式へ後方互換で読み込めるようにする。

### フェーズ1：保存の構造化
右クリック保存と手入力保存を、同じフラグメント保存の流れに統一します。

- **フラグメントベースの保存**: 右クリックで保存されるテキスト・画像・URLを共通形式で保存する。
- **Inboxの導入**: 保存直後は必ず `Inbox` に入れる。
- **Sourceの保持**: URL、タイトル、選択テキスト、画像元ページを Fragment に残す。

### フェーズ2：閲覧と整理
保存した情報を、見つけやすく並べ替えられるようにします。

- **タイムラインビュー**: 保存日時順に Fragment を表示する。
- **フィルタリング**: `text` / `image` / `url` などの種類で絞り込めるようにする。
- **Branch移動**: Fragment を `Inbox` から別の Branch に移す操作を追加する。

### フェーズ2の進捗
- **タイムラインビュー**: サイドバーの一覧で実装済み。
- **フィルタリング**: Branch と Type の絞り込みを実装済み。
- **Branch移動**: インライン編集UIから変更可能。

### フェーズ3：書く・まとめる
保存した Fragment を、日記として編集できる状態にします。

- **Markdown日記エディタ**: Fragment を本文に差し込める編集画面を作る。
- **Document管理**: 複数の Fragment を参照する Document を保存する。
- **エクスポート**: `.md` をダウンロードとして書き出す。

### フェーズ4：ローカル連携
ローカルファイルを扱う必要が出たら、この段階で拡張します。

- **フォルダ選択**: ユーザーが選んだディレクトリをルートとして扱う。
- **再帰的な閲覧**: サブディレクトリもたどって `.md` を読む。
- **保存先の固定化**: 必要なら Native Messaging を使って任意フォルダへ自動保存する。

### 直近の実装順
今すぐ着手するなら、次の順番が最短です。

1. `Fragment` 型と保存APIを追加する。
2. 既存 `memos` を `Fragment` として読む互換層を入れる。
3. `Inbox` をデフォルト Branch にする。
4. タイムライン表示を新スキーマに合わせて切り替える。
5. `.md` エクスポートを追加する。

## 4. データ構造の技術的詳細
将来的な検索や拡張性を考慮した設計です。

- **Fragment**: 保存の最小単位（ID, Type, Content, Source, BranchId）。
- **Branch**: 分類（ID, Name）。Fragmentを束ねる。
- **Document**: 上位の構造体。複数のFragmentを参照（IDで管理）し、Markdownテキストを保持する。
- **Inbox**: 新規保存のデフォルト先。最初はここに集約する。

### 推奨する保存イメージ
- `Fragment.id`: 一意ID
- `Fragment.type`: `text` / `image` / `url`
- `Fragment.content`: 本文または参照情報
- `Fragment.source`: 取得元URLやタイトル
- `Fragment.branchId`: 所属Branch
- `Fragment.createdAt`: 保存日時
- `Branch.id`: 一意ID
- `Branch.name`: 表示名
- `Branch.parentId`: 任意。階層化する場合に使う
- `Document.id`: 一意ID
- `Document.title`: 日記タイトル
- `Document.blockIds`: 参照するFragment ID群
- `Document.markdown`: 書き出し用Markdown本文

---

> [!TIP]
> **「自分だけの知識の断片をブラウザで集め、ローカル環境（Markdown）へとつないでいく」**
> これがSurfDiaryが目指す最終的なユーザー体験です。

# SurfDiary ロードマップ

このロードマップは、開発ブラウザを Vivaldi、利用者向けの基本互換を Chrome とする前提でまとめています。  
最終目標は、ブラウザ内で完結する文書ワークスペースを育て、将来的に画像付きの `.md` ファイルへきれいに書き出せるようにすることです。

## 現在の方針

- 開発確認は Vivaldi を主軸にする
- 利用者側の互換性は Chrome を基準に見る
- ブラウザ自体を、編集・時系列一覧・プレビューの作業空間として扱う
- アプリ状態と設定は `browser.storage.local` に保持する
- Vivaldi で使える範囲では File System Access API を活用する
- エクスポートの最終的な逃げ道として `downloads.download()` を残す
- `native messaging` は将来の高度なローカル同期用とし、初期の中核にはしない

## 検証手順

- 重要な変更のたびに Vivaldi で拡張機能を再読み込みする
- テキスト、URL、画像の保存内容が再起動後も残るか確認する
- 文書編集画面で編集できるか確認する
- Markdown の書き出しで、実際にローカルファイルが生成されるか確認する
- `SDAssets/` 出力と相対パスの画像リンクが保たれているか確認する
- 再読み込み後もプレビューとタイムラインが正しく描画されるか確認する

## プロダクト目標

- SurfDiary を単なる簡易メモではなくする
- テキスト、URL、画像、文書を再利用しやすい部品として整理する
- 収集用のタイムラインと、組み立て用の文書画面を両立させる
- オフラインでも開き直せる、きれいなローカルファイルを書き出す
- ブラウザ標準のメモ機能との差別化として、文書ワークフローを前面に出す

## フェーズ

### フェーズ 1: 収集

- テキスト、URL、画像をブラウザストレージへ保存する
- 収集内容をまとめるためのブランチ選択を支援する
- タイムラインは軽く、速く、見やすく保つ

### フェーズ 2: 閲覧

- ブロックをタイムライン形式で表示し、絞り込みできるようにする
- 保存済み内容をすばやく確認できるようにする
- 画像サムネイルは軽量に扱い、一覧の負荷を抑える

### フェーズ 3: 文書化

- 保存済みの部品から文書を組み立てられるようにする
- テキスト、URL、画像をドラッグ＆ドロップで文書に入れられるようにする
- 文書本文を直接編集できるようにする
- 生テキストだけでなく、画像やリンクが見えるプレビューを用意する

### フェーズ 4: エクスポート

- 文書をローカルの `.md` ファイルとして出力する
- 画像は Markdown ファイルの横にある `SDAssets/` フォルダへ出力する
- Vivaldi で安定して使えるエクスポート経路を優先する
- ブラウザが許す範囲で、選択したフォルダを再利用できるようにする

### フェーズ 5: ローカルワークスペース

- フォルダベースのワークスペースに対応する
- 1 つのディレクトリツリーの中で複数の `.md` ファイルを扱えるようにする
- 共有画像用の `SDAssets/` ディレクトリを維持する
- 必要ならフォルダ単位の索引を追加する

### フェーズ 6: ローカル同期

- ブラウザだけでは足りない場合に限って native messaging を追加する
- 自動同期、上書き、追記などの高度なローカル連携に使う
- 導入は任意にして、基本機能は追加インストールなしで動くようにする

## ドキュメント方針

- `TEXT`: テキスト本体のみを保存する
- `URL`: タイトルと URL をセットで保存する
- `IMAGE`: 閲覧用の軽いプレビューと、ローカル書き出し向けの元データを分けて考える
- `DOCUMENT`: 組み立て済み本文、部品参照、エクスポート用メタデータを保存する

## 保存方針

- 設定とアプリ状態は `browser.storage.local` に置く
- 文書内容は可能な範囲で軽量に保つ
- 長く残すべきローカルファイルは、ブラウザストレージではなくエクスポート結果として扱う
- 複数の出力先を増やしすぎず、1 つのワークスペースに寄せる

## 次にやること

- 文書仕様を Vivaldi 優先の方針に合わせて整理する
- エクスポート経路を Vivaldi で使いやすいファイル API に寄せる
- 画像の保存方針を、プレビュー・エクスポート・オフライン閲覧の観点で確定する
- 文書数が増えたときのために、ワークスペースフォルダ案を詰める
- 各段階の確認方法を明確にして、Vivaldi で開発確認しやすい状態を保つ
